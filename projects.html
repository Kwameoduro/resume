SkillDev — AI Skill
Development Platform

QA Engineering on an AI-powered platform that builds personalised learning paths for up to 5 user-selected skills.

The Problem

SkillDev's core engine — an AI-driven learning path generator — needed rigorous validation. Users input up to 5 skills and their self-assessed proficiency levels; the AI constructs a bespoke learning curriculum. The challenge: how do you write deterministic test cases for a non-deterministic AI output? And how do you ensure the surrounding platform — auth, skill selection UI, profile management, progress tracking — is bulletproof while the AI is being continuously retrained?

The Approach

I designed a two-layer test strategy. The first layer covered all deterministic platform functionality: user registration, skill selection (boundary testing for the 5-skill cap), proficiency level validation, and dashboard behaviour. These were automated using Selenium with Cucumber BDD, with Gherkin feature files authored alongside product owners to ensure test coverage reflected real user intent.

The second layer addressed the AI component using output-based validation: instead of asserting exact learning paths (impossible with generative AI), I defined guardrails — response time SLAs, content structure schemas, skill relevance scoring, and error handling under edge inputs (null skills, conflicting proficiency levels, unsupported skill categories). API tests via Rest Assured validated every contract between frontend and the AI backend.

Outcomes

✓

Identified and documented critical boundary defect — the 5-skill cap was enforced on the UI but not the API, allowing backend bypass attacks.

✓

Established the first AI output validation framework on the project — a schema-based approach now used by the full QA team.

✓

Achieved full traceability between business requirements, Gherkin scenarios, and Jira Xray test executions — zero orphaned test cases.

✓

Automated 80%+ of regression scenarios, reducing manual sprint regression time from days to hours.

SwagLabs — Full-Spectrum
Test Automation Suite

A comprehensive automation framework covering web, mobile, and security testing for the SwagLabs e-commerce application — built as a showcase of multi-layer QA engineering.

The Problem

SwagLabs (saucedemo.com) is a purpose-built test application designed to challenge QA engineers with real-world e-commerce scenarios — login flows, product listings, cart management, checkout sequences, and user role variations. The goal: build a production-grade automation framework from scratch that covers web, mobile, and security dimensions with a single coherent architecture, not three disconnected scripts.

The Approach

Architected a unified test framework using Selenium and Selenide for web, with Appium handling the mobile layer on Android. Both shared the same Page Object Model base classes, reducing code duplication to near zero. Test data was externalised into configuration files, enabling parameterised execution across all six SwagLabs user roles (standard, locked-out, problem, performance-glitch, error, visual).

Security analysis was layered on top using OWASP ZAP in both passive spider mode (crawl and observe) and active scan mode (probe for vulnerabilities). Findings were classified against the OWASP Top 10 and reported with severity ratings, CWE references, and recommended remediations. The entire suite was wired into a Jenkins pipeline with Docker-containerised execution and post-run reports pushed to Slack.

Outcomes

✓

Delivered a cross-platform framework — same test logic, executed on both web browser and Android mobile — a significant architectural achievement.

✓

Identified OWASP Top 10 vulnerabilities including insecure direct object references and insufficient transport layer security — documented with full remediation guidance.

✓

All 6 user role scenarios fully parameterised and automated — including the deliberately broken "problem user" and "performance-glitch user" edge cases.

✓

Zero manual intervention needed post-setup — Jenkins pipeline runs the full suite, generates reports, and notifies the team via Slack automatically.
